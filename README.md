# Mailing Manager — Django сервис рассылок

Веб-приложение для управления email-рассылками, клиентами, сообщениями и статистикой отправок. Проект разработан в учебных целях и демонстрирует работу с Django, системой прав пользователей, Redis-кешированием и email-инфраструктурой.

# Установка и запуск
1. Клонируем репозиторий
```
git clone <URL_репозитория>
cd <папка_проекта>
```
2. Устанавливаем зависимости

```pip install -r requirements.txt```

3. Создаем .env файл
```
cp .env.example .env
```

4. Применяем миграции

```python manage.py migrate```

5. Создаем суперпользователя

```python manage.py createsuperuser```

6. Создаем роли менеджеров

```python manage.py create_roles```

7. Запускаем сервер

```python manage.py runserver```

# Описание проекта

Функциональные возможности
### 1. Получатели рассылок

Добавление, просмотр, редактирование, удаление.

Поля: email, ФИО, комментарий.

Привязка к владельцу.

### 2. Сообщения

Управление шаблонами писем.

Поля: тема, тело письма.

### 3. Рассылки

Планирование отправок по времени: start_time, end_time.

Автоматический статус:

Создана

Запущена

Завершена

Выбор получателей и сообщений.

Ручной запуск рассылок.

Отправка через сервис и через кастомную команду:

python manage.py send_mailing

### 4. Попытки рассылок (лог)

Статус попытки: успешно / неуспешно.

Время попытки.

Ответ сервера.

Привязка к конкретной рассылке.

### 5. Пользователи

Регистрация с подтверждением email.

Авторизация.

Сброс пароля.

Просмотр и редактирование профиля.

Менеджеры могут:

Просматривать всех пользователей,

Блокировать пользователей,

Принудительно останавливать рассылки.

### 6. Права доступа

Пользователь управляет только своими сущностями.

Менеджер — только просмотр, без редактирования.

Кастомные права в моделях и проверка в шаблонах.

### 7. Кеширование

Кеширование представлений через cache_page.

Поддержка Redis:

LOCATION=redis://127.0.0.1:6379/1

## Полезные команды:

### Отправка рассылок вручную
```python manage.py send_mailing```

### Создание ролей менеджеров
```python manage.py create_roles```

### Вход в shell
```python manage.py shell -i ipython```

Структура проекта

mailing/        # Рассылки, сообщения, получатели, лог отправок

users/          # Регистрация, вход, профиль, ACL, менеджеры

config/         # Настройки Django

templates/      # HTML-шаблоны

static/         # Статические файлы, CSS, Bootstrap
